# AirSketch — Academic Presentation Guide

> **For a future Claude session.** This file tells you *how* to build a detailed
> academic presentation for AirSketch. The *content* lives in
> `PROJECT_COMPENDIUM.md` (read it first). The prior deck is
> `AirSketch_MidProject_v2.pptx` — this new deck must be **more detailed** and
> include **everything new since then** (see Compendium §16).

---

## 1. Goal, audience, format

- **Goal:** a polished, detailed **academic** presentation of AirSketch suitable for
  a course / project defense / research seminar.
- **Audience:** technically literate — course staff and peers. Assume they know basic
  ML/CV but not MediaPipe/OpenVINO internals; explain those.
- **Length:** ~16–22 slides (the prior deck was ~7; this is the "more detailed" version).
- **Output format:** a `.pptx` built with **`python-pptx`** (the environment's npm /
  pptxgenjs path is proxy-blocked — `python-pptx` is the proven tool here). Name it
  `AirSketch_Academic_v1.pptx` (don't overwrite the mid-project decks).

## 2. Style rules (match + elevate the prior deck)

- **Low text on slides; talk, don't read.** Few bullets, large type, one idea per slide.
- **Speaker notes carry the detail.** The prior deck used **Hebrew speaker notes** —
  keep Hebrew notes unless the user says otherwise (confirm language preference at start).
- **Clean, consistent, professional** visuals; a uniform light theme; a small accent
  color. Use diagrams over paragraphs. (If the user wants Check Point branding, the
  `anthropic-skills:checkpoint-brand` skill exists; for a neutral academic look, a
  simple uniform theme is fine — ask.)
- Use the project accent identity if desired: dark panels + aqua/teal accent (the app's
  `airsketch.ico` and HUD palette), so slides visually echo the product.
- Every slide: a **title**, a **minimal visual/bullets**, and **rich speaker notes**.

## 4. Assets / figures to (re)generate

- **Architecture diagram** (§3) and **hand-tracking pipeline diagram** (§5) — draw with
  python-pptx shapes or a small matplotlib/PIL render.
- **Hand-landmark overlay images** (good vs bad normalization) — can be regenerated by
  running `OpenVINOHandTracker` on a still and saving the overlay (see how it was done
  in the session: render to PNG, crop/zoom).
- **FPS bar**, **CNN accuracy**, **AI-stack table** — from §4, §11.
- **Launcher / Hebrew-UI screenshots** — capture from a live run, or render samples
  (the Hebrew renderer can draw sample UI to a PNG, as was done during development).
- Reuse the app **icon** (`airsketch.ico` / `airsketch_icon_preview.png`).
