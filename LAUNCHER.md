# AirSketch Launcher

A small GUI front-end (`launcher.py`) that turns the app's CLI flags into a
friendly feature-flags screen, then starts AirSketch the way you chose.

## Run from source (no build needed)

```powershell
python launcher.py
```

Pick your options, watch the live **Launch command** preview, click **Launch ▶**.
The launcher closes and the app starts in a separate process.

## What the GUI exposes

| Group | Options |
|---|---|
| **Mode** | Classroom challenge game / Notebook (freehand) · Theme (geometry/objects/mixed) · **Language (en/he)** |
| **Camera** | Index · Mirror (selfie) · Rotate 180° (on by default for the VDI camera) |
| **AI backends & devices** | Hand tracking backend (**openvino** default / mediapipe) · Hand device (AUTO/CPU/GPU/NPU) · Hand-tracking debug overlay · CNN device |
| **Features** | Voice commands · Teacher voice/dictation · Whiteboard capture (PP-OCR) · Understand board (LLM) · Qwen2-VL titling (notebook) · Snap-to-shape · Disable CNN |

Everything maps 1:1 to `airsketch.main` CLI flags — the preview box shows the exact
command, and a unit test (`tests/test_launcher.py`) asserts every emitted flag parses.

## Build the single-file exe

```powershell
pip install -e .[exe]      # installs pyinstaller
python make_icon.py        # (optional) regenerate airsketch.ico
build_exe.bat              # -> dist\AirSketch\AirSketch.exe  (+ a reusable AirSketch.spec)
```

The exe icon is `airsketch.ico` (generated by `make_icon.py`; `build_exe.bat` passes
`--icon airsketch.ico`).

The build is **`--onedir`**: it produces a *folder* `dist\AirSketch\` containing
`AirSketch.exe` plus its dependencies (`_internal\`). **Keep the folder together** and
run `AirSketch.exe` from inside it. The exe is both the launcher *and* the app runner
(it re-invokes itself with `--launch-app <flags>`), so no Python install is needed on
the target.

### Notes / honest limitations
- **Why `--onedir`, not `--onefile`:** a single-file exe unpacks ~250 MB to
  `%TEMP%\_MEIxxxxx` on every launch. On locked-down machines (e.g. a Check Point VDI
  with endpoint protection) the antivirus can corrupt that temp extraction mid-launch,
  causing a `pyi_rth_pkgres` / `base_library.zip not found` crash. `--onedir` has no
  temp extraction, so it avoids that and also starts much faster. The trade-off is a
  folder instead of one file.
- **Models are NOT bundled** (they are several GB). The app finds `models\` by walking
  up from the exe's location (`resolve_root`), so running it from inside the project
  tree just works. To ship standalone, put a `models\` folder beside `AirSketch.exe`.
- Hidden run modes used by the build/launcher (not for end users):
  `AirSketch.exe --launch-app <flags>` runs the app directly;
  `AirSketch.exe --check` is an import-only smoke test used to verify a build.
- `torch`/`torchvision` (training-only) are excluded from the exe to keep it lean.
